## Overview

BugSense was built in **5 weeks** as part of an internal challenge at **Calda**. The team went from a Slack message to a fully working product with a dashboard, two SDKs, a ChatGPT integration, and full Linear/Slack connectivity.

---

## Week 1 — Brainstorming

**Feb 17 – Feb 23**

- **Feb 17** — Challenge kickoff via Slack. Each team member brainstormed individually: pain points, scope suggestions, and extra-mile ideas.
- **Feb 23** — First team meeting. Everyone presented their brainstorming results. The team created a feature list, decided on the direction (dashboard + SDKs + AI integrations), and assigned roles.

---

## Week 2 — Planning & SDK prototypes

**Feb 24 – Mar 2**

- Continued planning and aligning on the feature set.
- **Web SDK** — initial project scaffolding, screenshot capture, device info collection.
- **Flutter SDK** — first commit, project structure setup.

---

## Week 3 — Design & early development

**Mar 3 – Mar 9**

- **Mar 4** — Second team meeting. Finalized the feature plan, defined the notification system, reviewed UX examples from the designer.
- **Mar 8** — First version of the UI design ready for review.
- **Web SDK** — redesigned report sheet based on Figma designs, added improvements.
- **Flutter SDK** — device info, screen recording, report sheet redesign from Figma.
- **Backend** — initial commit, Supabase setup, user management tables.

---

## Week 4 — Full-stack build

**Mar 10 – Mar 16**

- **Mar 10** — UI design review with the team. Development started across all repos.

**Backend:**
- Email and Slack notification services
- Branded auth email templates (invite, confirmation, password recovery)
- Linear integration — project linking, issue creation, bidirectional sync
- Linear webhooks — issue and comment events
- Comments CRUD endpoints
- Slack digest job (new ticket notifications every 30 min)
- AI duplicate detection and weekly report generation
- Comment email digest
- ChatGPT app endpoints and tools

**Frontend:**
- Auth flows (signup, login, set password)
- Client dashboard with kanban board
- Admin dashboard — projects table, create project, Linear connection
- Weekly reports UI (view, edit, approve)
- Team leads management page

**Web SDK:**
- Complete design from Figma, Supabase auth, login screen, settings
- Connected to real backend API

**Flutter SDK:**
- Redesigned UI from Figma, markdown support
- Connected to real backend API

---

## Week 5 — Polish & ship

**Mar 17 – Mar 20**

**Backend:**
- Interactive Slack webhooks
- ChatGPT OAuth flow and MCP server
- Inbox notifications system
- Team leads CRUD
- Linear OAuth refactoring and multi-user support
- Duplicate issue prevention
- Hotfixes and stabilization

**Frontend:**
- Client UI refinements
- API endpoint updates

**SDKs:**
- API call fixes, video upload improvements

**Docs:**
- Full documentation site created
- Architecture, authentication, data model, integration guides
- API reference for all endpoints
- SDK documentation
- Testing guide

**Mar 20** — Deadline. App submitted.
