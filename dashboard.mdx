## Overview

The BugSense dashboard is a React single-page application that serves as the primary interface for both **admins** (team leads) and **clients**. Admins manage projects, connect Linear, invite clients, and review AI-generated weekly reports. Clients view their issues on a kanban board, submit new bugs, leave comments, and read weekly summaries.

**Live app:** [calda-bugsense-frontend.vercel.app](https://calda-bugsense-frontend.vercel.app/)

## Authentication flow

1. User logs in via Supabase REST endpoint (`POST /auth/v1/token`).
2. JWT and refresh token are stored in `localStorage`.
3. All backend requests include `Authorization: Bearer <token>`.
4. On 401 response, tokens are cleared and the user is redirected to `/login`.

### Invite flow

1. Admin invites a client via email through the Create Project form.
2. Client receives an invite email with a link containing `#access_token=...&type=invite`.
3. The app detects the hash fragment and redirects to `/set-password`.
4. Client sets their password and is redirected to `/login`.

### Password recovery

1. User requests a reset on `/forgot-password`.
2. Email contains a link with `#access_token=...&type=recovery`.
3. Same `/set-password` page handles the reset.

## Key features

### Kanban board

Issues are displayed in a kanban board grouped by status columns. Clicking a ticket opens a detail sidebar with the full description, metadata, attachments, and a comment thread that syncs with Linear.

### Create ticket (client)

Clients can create new issues with:
- Title and rich-text description (bold, italic, strikethrough)
- Platform selector (Web, Apple, Android, Figma)
- Environment (Staging, Production)
- File attachments

New issues are automatically synced to Linear with a "Client view" label.

### Weekly reports

Every Monday, the backend generates AI-powered reports per project. The Reports page shows these grouped by month. Admins can edit report content and approve reports before they are visible to clients.

### Inbox

The Inbox page shows real-time notifications for project activity. Notifications are triggered by:

- **New comments** — when a comment or reply is added to an issue in the client's project
- **Task completed** — when an issue's status changes to done
- **New report** — when a weekly report is approved and published

Each notification links to the relevant issue or report. Users can mark individual notifications as read or mark all as read.

### Linear integration

Admins connect their Linear workspace via OAuth directly from the sidebar. Once connected, projects can be linked to Linear projects during creation, and all issues and comments sync bidirectionally.

## Environment variables

| Variable | Description |
|----------|-------------|
| `VITE_SUPABASE_URL` | Supabase project URL (auth REST endpoints) |
| `VITE_SUPABASE_ANON_KEY` | Supabase publishable anon key |
| `VITE_API_BASE_URL` | Backend API URL (not needed in dev — Vite proxies `/api`) |

## Local development

```bash
git clone <repo-url> calda-bugsense-frontend
cd calda-bugsense-frontend
npm install
npm run dev
```

The Vite dev server proxies `/api` requests to the backend at `https://calda-bugsense-backend.onrender.com`, so you don't need to set `VITE_API_BASE_URL` locally.
