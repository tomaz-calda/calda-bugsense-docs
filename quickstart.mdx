## Prerequisites

- [Node.js](https://nodejs.org) v20+
- [Supabase CLI](https://supabase.com/docs/guides/cli) for local development
- A [Linear](https://linear.app) account (for integration testing)
- [Docker](https://www.docker.com) app for local Supabase development

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url> calda-bugsense-backend
    cd calda-bugsense-backend/app
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Start local Supabase">
    From the repository root (not `app/`):

    ```bash
    supabase start
    ```

    This spins up a local PostgreSQL instance on port `54322`, Supabase Auth, and Storage.
  </Step>
  <Step title="Configure environment variables">
    Copy the example env file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    At minimum, set these variables for local development:

    | Variable | Description |
    |----------|-------------|
    | `DATABASE_URL` | `postgresql://postgres:postgres@127.0.0.1:54322/postgres` |
    | `SUPABASE_URL` | `http://127.0.0.1:54321` |
    | `SUPABASE_ANON_KEY` | From `supabase status` output |
    | `SUPABASE_SERVICE_ROLE_KEY` | From `supabase status` output |

    See [environment variables](#environment-variables) for the full list.
  </Step>
  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    The API server starts on `http://localhost:8000`. Hit the health endpoint to confirm:

    ```bash
    curl http://localhost:8000/api/v1/health
    ```

    You should see `{"status":"ok"}`.
  </Step>
  <Step title="Import /api/ into Bruno (optional)">
    ```bash
    bruno import /api/
    ```

    This will import the API endpoints into Bruno for testing.
  </Step>
</Steps>

## Available scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start development server with hot reload |
| `npm run build` | Compile TypeScript to `dist/` |
| `npm run start` | Run the compiled production build |
| `npm run db:generate` | Generate Drizzle migration files |
| `npm run db:push` | Push schema changes to the database |
| `npm run db:studio` | Open Drizzle Studio for database inspection |

## Environment variables

| Variable | Description |
|----------|-------------|
| `PORT` | Server port (default: `8000`) |
| `DATABASE_URL` | PostgreSQL connection string |
| `SUPABASE_URL` | Supabase instance URL |
| `SUPABASE_ANON_KEY` | Supabase anonymous key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (for admin operations) |
| `SUPABASE_PROFILE_BUCKET` | Storage bucket name for profile images |
| `BREVO_API_KEY` | Brevo (Sendinblue) API key for sending emails |
| `BREVO_DEFAULT_SENDER_EMAIL` | Default sender email address |
| `BREVO_DEFAULT_SENDER_NAME` | Default sender display name |
| `SLACK_BOT_TOKEN` | Slack bot OAuth token |
| `LINEAR_CLIENT_ID` | Linear OAuth application client ID |
| `LINEAR_CLIENT_SECRET` | Linear OAuth application client secret |
| `LINEAR_OAUTH_REDIRECT_URI` | OAuth callback URL |
| `LINEAR_OAUTH_SCOPES` | OAuth scopes (default: `read,write,admin`) |
| `OPEN_API_KEY` | OpenAI API key for AI features |
| `OPENAI_DEDUP_MODEL` | Model for duplicate detection (default: `gpt-4o-mini`) |
| `OPENAI_WEEKLY_REPORT_MODEL` | Model for weekly reports (default: `gpt-4o-mini`) |
| `CRON_SECRET` | Shared secret for cron job authentication |
| `CLIENT_INVITE_REDIRECT_URL` | URL clients are redirected to after accepting an invite |
| `BASE_URL` | Public base URL for the API (used in webhooks) |
| `FRONTEND_URL` | Frontend application URL |
