---
title: "Update team lead"
api: "PATCH /api/v1/team-leads/{teamLeadId}"
description: "Updates a team lead's profile information. Admin only."
---

## Authentication

<Icon icon="shield" /> Requires **admin** role

## Path parameters

<ParamField path="teamLeadId" type="string" required>
  UUID of the team lead to update.
</ParamField>

## Request body

At least one field is required.

<ParamField body="firstName" type="string | null">
  First name (1–100 characters). Pass `null` to clear.
</ParamField>

<ParamField body="lastName" type="string | null">
  Last name (1–100 characters). Pass `null` to clear.
</ParamField>

<RequestExample>
```json
{
  "firstName": "Janet",
  "lastName": "Smith-Jones"
}
```
</RequestExample>

## Response

<ResponseField name="data" type="object">
  The updated team lead profile.

  <Expandable title="team lead object">
    <ResponseField name="id" type="string">User UUID</ResponseField>
    <ResponseField name="email" type="string">Email address</ResponseField>
    <ResponseField name="firstName" type="string | null">First name</ResponseField>
    <ResponseField name="lastName" type="string | null">Last name</ResponseField>
    <ResponseField name="profileImagePath" type="string | null">Avatar storage path</ResponseField>
    <ResponseField name="active" type="boolean">Whether the account is active</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp</ResponseField>
    <ResponseField name="role" type="string">`admin`</ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json 200
{
  "data": {
    "id": "660e9500-f30c-52e5-b827-557766550000",
    "email": "jane@agency.com",
    "firstName": "Janet",
    "lastName": "Smith-Jones",
    "profileImagePath": null,
    "active": true,
    "createdAt": "2026-01-15T10:00:00.000Z",
    "updatedAt": "2026-03-19T12:00:00.000Z",
    "role": "admin"
  }
}
```
</ResponseExample>

## Notes

- The target user must have the `admin` role. Returns `404` if the user is not a team lead.
