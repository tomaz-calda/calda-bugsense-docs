## Base URL

All API endpoints are served under:

```
https://<your-domain>/api/v1
```

For local development, this is `http://localhost:8000/api/v1`.

## Authentication

Most endpoints require a valid JWT token in the `Authorization` header:

```
Authorization: Bearer <jwt_token>
```

Tokens are issued by Supabase Auth. See the [authentication guide](/authentication) for details.

Some endpoints require **admin** role access. These are noted with <Icon icon="shield" /> in the endpoint documentation.

## Response format

All successful responses follow this structure:

```json
{
  "data": { ... }
}
```

List endpoints return an array:

```json
{
  "data": [{ ... }, { ... }]
}
```

## Error responses

Errors return an appropriate HTTP status code with a JSON body:

```json
{
  "error": "Human-readable error message"
}
```

Validation errors include field-level details:

```json
{
  "error": "Invalid request payload.",
  "details": {
    "fieldErrors": {
      "title": ["Required"]
    }
  }
}
```

## Common status codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Resource created |
| `400` | Bad request (validation error) |
| `401` | Unauthorized (missing or invalid token) |
| `403` | Forbidden (insufficient role) |
| `404` | Resource not found |
| `500` | Internal server error |

## Rate limiting

There is currently no rate limiting enforced at the application level. If deployed behind a reverse proxy or API gateway, rate limiting may be configured there.
