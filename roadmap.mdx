<Note>
  This is a living document. Items listed here are planned but not yet scheduled. The list will grow over time.
</Note>

## Notifications

**Client notification settings** 
  - configure when and how often clients receive notifications
  - New comments on their issues

**Team lead notification settings** 
  - configure when and how often team leads receive notifications
  - New issues reported by clients

## Reports

- **Configurable report generation** — control when and how often weekly reports get generated

## SDKs

- **Platform coverage** — create SDKs for all platforms we use (beyond the current Flutter and Web SDKs)
- **AI-powered input assistance** — autosuggestion and autocomplete inside the SDK to help users automatically write issue descriptions
- **Instrumentation guides** — instructions and prompts so that logs for events the SDK reads can be added to any Flutter, Next.js, or other platform

## AI & Integrations

- **GitHub / Figma MCP integration** — connect GitHub repos and Figma files via MCP so that AI can suggest fixes based on client issue and feature reports
- **Feature requests** — dedicated feature request workflow (separate from bug reports)

## Security

- **Two-factor authentication (2FA)** — add 2FA support for user accounts

## Slack improvements

- **Channel listing** — list available team lead Slack channels instead of requiring the channel ID to be entered manually
- **Private channel support** — allow inviting the Slack bot to private channels

## Team & Operations

- **On-call developers** — support for assigning and managing on-call dev rotations
