## Overview

Linear is the core integration in BugSense. It provides bidirectional syncing of issues, comments, and attachments between BugSense and your Linear workspace.

## Connection flow

BugSense uses OAuth 2.0 to connect to Linear:

<Steps>
  <Step title="Initiate OAuth">
    An admin calls `GET /api/v1/integrations/linear/connect` which returns an authorization URL.
  </Step>
  <Step title="Authorize in Linear">
    The admin is redirected to Linear's OAuth consent screen and grants access.
  </Step>
  <Step title="Callback">
    Linear redirects back to `GET /api/v1/integrations/linear/callback` with an authorization code. BugSense exchanges this for an access token, stores it, and redirects the user to the frontend at `/app/linear/connected`.
  </Step>
  <Step title="Webhook registration">
    BugSense automatically registers webhooks on the connected Linear team to receive events.
  </Step>
</Steps>

## What syncs

### BugSense → Linear

| Event | Action in Linear |
|-------|-----------------|
| Client creates an issue | New Linear issue created with "BugSense - Client View" label. Platform, environment, and app version metadata are appended to the issue description |
| Client creates a duplicate issue | "BugSense - Possible Duplicate" label is automatically applied |
| Client adds a comment | Comment mirrored to the Linear issue |

### Linear → BugSense (via webhooks)

| Linear event | Action in BugSense |
|-------------|-------------------|
| Issue updated (status, priority, assignee) | Local issue record updated |
| Comment created | Comment stored locally |
| Comment updated | Local comment updated |
| Attachment added | Attachment reference stored |
| Project updated | Project name synced |

## Issue metadata

When BugSense creates an issue in Linear, it appends a metadata block to the issue description:

```
---

**Platform:** web
**Environment:** production
**App Version:** 1.2.3
```

This metadata is only added when at least one of platform, environment, or app version is present.

## Labels

BugSense manages two special labels on Linear teams:

| Label | Color | Purpose |
|-------|-------|---------|
| **BugSense - Client View** | `#4B7BEC` | Applied to issues visible to clients in the BugSense dashboard |
| **BugSense - Possible Duplicate** | `#a20000` | Automatically applied when an issue is flagged as a potential duplicate |

Labels are created automatically on the Linear team if they don't already exist.
